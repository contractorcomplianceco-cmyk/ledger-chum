# Project APEX — Vision

LedgerOS becomes **The Financial Operating System for Growth Companies**.

We stop shipping module → table → report. We ship **signal → explanation → decision → action → outcome**.

## The ten decision questions leadership must answer

1. Can we safely spend money?
2. Did we make money?
3. Why did profit change?
4. What requires attention today?
5. Where are we leaking revenue or margin?
6. What is financially unhealthy?
7. Can we hire?
8. Can we expand?
9. Can we afford a proposed decision?
10. What is the smartest next action?

Every APEX surface must trace back to one of these questions and answer it with evidence, confidence, freshness, a recommended action, and the required approval path.

## Non-negotiables

- No backend, no Supabase, no auth changes are introduced by APEX work.
- Existing routes remain functional; operational navigation remains the default.
- Executive navigation is an opt-in overlay persisted to `localStorage`.
- All AI is advisory; every AI response carries persona, evidence, confidence, freshness, assumptions, missing data, action, approval, and a demonstration label.
- All financial impact numbers on APEX surfaces are demonstration data.

## Visual direction

Approximately 70% light workspace / 30% dark executive surfaces. Deep navy sidebar. Cool-gray page background. White operational cards. Dark navy intelligence cards used selectively. Electric blue, cyan, teal, and violet accents. Refined gradients, fine borders, soft depth, controlled glow. Reduced motion respected everywhere.
